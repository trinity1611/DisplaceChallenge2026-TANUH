## Medical Topic Identification (Track 3 – DISPLACE-M)

This repository provides a **baseline implementation for Medical Topic Identification** from real-world healthcare conversations collected in rural India. The system is designed to process **multilingual conversational speech transcripts (Hindi, Hinglish, and code-mixed)** and extract the patient’s **current health problems** using a fine-tuned large language model.

## Track Information

This repository is the official baseline for:

### **Track 3 – Medical Topic Identification**

**Task:**  
Automatically identify and extract the **patient’s ongoing and current health problems** from healthcare conversations.

The output is a **comma-separated list of medical problem topics** for each conversation.

The pipeline integrates:
- **LLM-based medical topic extraction (Qwen - Qwen/Qwen2.5-3B-Instruct)**
- **Hindi → English translation (LLaMA-meta-llama/Llama-3.2-3B-Instruct)**
- **Semantic evaluation using BERTScore and Lexical overlap using ROUGE-1**

## 2. Setup Instructions

### Step 1: 

```
# Clone Repository

git clone https://github.com/displace2026/DISPLACE-2026-Baselines.git

cd DISPLACE-2026-Baselines/Track3_TI
```

### Step 2:

- The TI baseline **requires Automatic Speaker Recognition (ASR) outputs** as input.
- Track1_SD and Track2_ASR needs to set-up and executed to get ASR output.
- To run **Track 1 – Speaker Diarization (SD)**, update the `run.sh` as follows:
     - `data_dir_path ='../Track3_TI/data/Track_3_TI_DevData_1'`
     - `dir_containing_files='Hindi'`
     - `wav_dir='Audio'`
        
    (Adapt the first two fields as per the track and language accordingly)
- **Run the SD baseline** by following the instructions provided in the `Track1_SD` README. The per-file rttms will be generated at `Track1_SD/data_directory/gen_rttm` 
- To run **Track2 - ASR**, update the ```config/asr.yaml``` as follows:
     - `input_audio_folder ='../Track3_TI/data/Track_3_TI_DevData_1/Hindi/Audio'`.
- **Run the ASR baseline** by following the instructions in the README of Track2_ASR. 
- Update the input path field `input_path` in TI configuration file ```config/config.yaml``` to point to the **output ASR transcripts generated by the ASR system**.


```
# Install Dependencies
# The same environment created for Track 2 - ASR will work for Track 3 - TI.
# To the Track 2 - ASR environment, install additional dependencies specific to Track3_TI using the command below:

pip install -r requirements.txt
```
### Step 3:

```
# Set HuggingFace Token (For LLaMA Translation)

export HF_TOKEN=your_huggingface_token
```

Please follow the below instructins for Model Setup.

### Model Details
| Feature | Details |
| :--- | :--- |
| **Model Name** | `Llama-3.2-3B-Instruct` |
| **Repository** | [meta-llama/Llama-3.2-3B-Instruct](https://huggingface.co/meta-llama/Llama-3.2-3B-Instruct) |
| **Parameters** | 3B |
| **Reference** | [Dubey et al., 2024](https://arxiv.org/abs/2407.21783) |

### Model Setup
| Step | Action |
| :--- | :--- |
| **1. Request Access** | Go to the [Hugging Face model page](https://huggingface.co/meta-llama/Llama-3.2-3B-Instruct) and request access. |
| **2. Generate Token** | Generate a **Hugging Face Token** from your account settings. |
| **3. Configure** | Update the `hf_token` field in `configs/summarization.yaml` with your token. |

## 4. Infernece

To run the TI baselines please follow below steps. 

```
python src/run_topic_inference.py
```

> **Note:** The identified topics and corresponding metrics will be saved in ```outputs/Prediction_topic/``` and,  ```outputs/metrics/```
