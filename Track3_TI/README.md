## Medical Topic Identification (Track 3 Topic Identification)

This repository provides a **baseline implementation for Medical Topic Identification** from real-world healthcare conversations collected in rural India. The system is designed to process **multilingual conversational speech transcripts (Hindi, Hinglish, and code-mixed)** and extract the patient’s **current health problems** using a fine-tuned large language model.

The pipeline integrates:
- **LLM-based medical topic extraction (Qwen + LoRA)**
- **Hindi → English translation (LLaMA-based)**
- **Semantic evaluation using BERTScore**
- **Config-driven execution for reproducibility**

This work is inspired by the DISPLACE-M challenge and focuses on **semantic understanding of medical complaints**, rather than simple keyword extraction.

---

## Track Information

This repository is the official baseline for:

### **Track 3 – Topic Identification**

**Task:**  
Automatically identify and extract the **patient’s ongoing and current health problems** from healthcare conversations.

The output is a **comma-separated list of medical problem topics** for each conversation.

## 3. Infernece

To run the TI baselines please follow below steps. 

### Prerequisites

- The TI baseline **requires Automatic Speaker Recognition (ASR) outputs** as input.
- Change the config file of the Track2_ASR system `input_audio_folder` field with the path of the Track3_TI audio file path and make the `input_gt` field as `None`.
- **Run the ASR baseline before TI inference.** by following the instructions in the README of Track2_ASR. 
- Update the input path field `input_path` in TI configuration file ```config/config.yaml``` to point to the **output ASR transcripts generated by the ASR system**.

```
# To run Track 2 - ASR Pipeline
python ../Track2_ASR/scripts/asr.py
```

```
# Then run the Topic Identification Pipeline

python src/run_topic_inference.py
```

> **Note:** The identified topics and corresponding metrics will be saved in ```outputs/Prediction_topic/``` and,  ```outputs/metrics/```

## 4. Evaluation Metrics

The following metrics will be used for evaluating Track 4 - Topic Identification task. 

### Extractive Score
In extractuve score, we are measuring the no. of topics correctly identified by using a similarity based Thresholding, and the similarity are being measured using ```all-MiniLM-L6-v2``` model.

- Precision

- Recall

- F1 Score

### BERTScore

Computes similarity using contextual embeddings (BERT), focusing on semantic meaning rather than exact word overlap.
